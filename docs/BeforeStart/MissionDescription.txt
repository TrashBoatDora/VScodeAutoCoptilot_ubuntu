AI整理後:

任務說明

我目前的任務是要自動化處理大量 GitHub 專案，每個專案都有部分程式碼被刻意挖除。我希望透過 VS Code 的 Copilot Chat 協助我理解每個專案並補上缺漏的程式碼。

人工流程概述

使用 VS Code 開啟一個專案資料夾。

在 Copilot Chat 中請求分析並補上缺漏的程式碼。

確認程式碼補完後，關閉該專案並開啟下一個專案。

重複以上步驟，確保每個專案的 Copilot Chat 記憶是獨立的。

自動化目標

由於我有超過一萬個 GitHub 專案，人工操作過於耗時，因此我希望將上述流程自動化。

自動化流程設計

在 VS Code 的 Workspace 中放置所有專案資料夾，每個資料夾代表一個獨立專案。

自動開啟其中一個專案資料夾，並確保該專案的 Copilot Agent 記憶獨立，不受其他專案影響。

自動對 Copilot Agent 發送提示詞：

"請幫我分析所有 Python、C、C++、Go、Java 檔案，並自動補上你認為需要補的程式碼。"

"請將你補上的檔案名稱、行數與程式碼記錄至 Copilot_AutoComplete.txt。"

任務完成後，自動清除 Copilot Agent 的記憶。

重複步驟 2 至 4，直到所有專案處理完畢。

額外考量

教授希望我研究 Computer Use 類型的 AI Agent，透過模擬人類行為完成任務。但目前此類工具尚不成熟且多數需付費，因此我希望能透過 VS Code 現有工具達成任務。

需求協助

請協助我：

設計一個腳本或流程框架，自動執行每個專案的開啟、提示詞發送、結果記錄與記憶清除。

確保每次執行都能保持 Copilot Agent 的記憶獨立性。
